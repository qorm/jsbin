- **[2026-07-16 结构解锁] 闭包属性侧表(`fn.x = 1`)+ 函数声明稳定身份(`f === f`)**:jsbin 函数是闭包/裸函数指针、**无属性容器**(对象头的 props_ptr@32/flags_ptr@40),故 `fn.x=1` 此前静默无效。**Phase 1 可行性**:对象头的 props/flags 侧表模型**不能直接搬进闭包头**——闭包(`16+cap*8`,magic@0/fnptr@8/captured@16+)与数组(32B)无固定空位,插槽须移位热字段,且闭包形态异构(用户闭包/bound/数组方法闭包/promise tramp/生成器共享 `{magic,fnptr,@16}` 但 @16 语义各异)→ 移位牵动热调用/捕获约定,风险高。**但 props 侧表可行**:GC 是保守**非移动** mark-sweep → 裸指针键稳定;数据段链表注册表(仿 `_symbol_registry`,`_data_gc_end` 前 → GC 根)保活挂的 props 对象。**不改闭包/数组布局 → 闭包逐字节不变 → 自举不受结构影响**。**Phase 2 实现(静态可解析切片)**:运行时 `_closure_props_registry`(数据根)+ find/ensure + `_closure_prop_get/set`(函数 props = 普通 jsbin 对象,存 24B 节点 {fn 裸指针,props,next})。编译器在**接收者静态解析到函数**(非类——类走静态成员机制,排除类修了首版类静态回归)时把 `fn.x` 读/写路由到侧表(复用 name/length 的 `_resolveFnNode`)。**函数声明身份**:声明作值此前每次引用新 alloc 闭包(`f===f` 为 false,且破坏侧表按指针键)→ 改为 memoize 到惰性全局槽 `_funcclosure_<symbol>`(GC 根,首建复用)→ `f===f` 真、声明上 `fn.x` 往返生效。**已工作(g1 与 node 逐字节)**:函数声明与 const 绑定函数字面量(箭头/函数表达式)上的 `fn.x` 置/改/读、缺属性→undefined、`f===f`/别名身份、与 name/length 反射共存。**有界泄漏**(仅用过 props 的函数被钉住,库多挂具名函数,量小)。gen1==gen2==gen3 逐字节(13664272,memoize-声明路径 cli.js 罕发故尺寸不变)、4 目标、fixtures 265/2、matrix 19/19。fixture `es/function-custom-properties`。**未覆盖(下一切片:`_object_get/_object_set` 运行时按 magic 路由)**:非静态可知的函数值——调用结果绑定(`const a=factory()`)、运行时值形参、别名写(`const g=f; g.x=1`)、`g.count++`(UpdateExpression 成员路径);数组 props(custom-tag `.raw`)属更后切片。
- **[2026-07-16 修真] 求值顺序三连修:成员赋值「对象→键→值」/ 对象字面量计算键「键→值」/ 复合赋值基与键只求值一次**:差分 sweep 定点的三个 ES 求值序分歧(侧效可观测)。①**成员赋值/下标读**:`o()[k()]=v()` 此前键先于对象(嵌套 `o(1)[k(1)][k(2)]=1` 全反序);修:赋值动态下标路 + 成员**读**三路(串动态索引/int 内联快路/通用 `_subscript_get`)按规范序发射(对象先、键后)。②**对象字面量计算键**:`{[k()]:v()}` 此前 value 先求;修:键先求值并**当场 ToPropertyKey 规范化**(`_valueToStr`/`_js_prop_key` 可触发用户 toString,亦须先于 value——`{[objWithToString]:v()}` 序对齐 node),规范化后回存槽再求 value。③**复合赋值双求值**:`o().v+=x`/`a[i++]+=1` 的脱糖 `member=(member OP rhs)` 令基/键各求两次;修:非纯基/键先各求值一次存帧槽,以 `__WithPrecomputed` 构造 preMember 再脱糖(复用逻辑复合赋值机制)。**自举安全设计(关键)**:全部改动以 `isPureExpr`(标识符/this/字面量)门控——**两操作数皆纯时顺序不可观测,保留既有发射序**(编译器自身 `a[i]`/`this.x+=…` 站点 codegen 不变),仅非纯站点走规范序;本 tree 布局下 gen1==gen2==gen3(13828112)且尺寸不变。读侧键值互扰边角(键求值改绑对象变量/对象求值改键变量)、ToPropertyKey 序、`a[i++]+=` 键一次、嵌套矩阵/位/串复合全对齐 node;smoke、fixtures 273/2。fixture `es/eval-order-member-literal`。**记录**:`fn.apply` 5 参 ABI 等既有偏差不在本修范围。

- **[2026-07-16 新特性] 用户函数/类的 `.name` / `.length` 反射(访问点静态解析)**:jsbin 函数是闭包/裸函数指针、**无属性容器**(故 `fn.x=1` 不生效),`foo.name`/`foo.length` 此前恒 undefined/0。不改闭包表示,改在**访问点**用编译期已知的 AST 元数据解析——`compileMemberExpression` 加门控分支,仅当接收者静态解析到用户函数/类/方法才发,其余 `.name`/`.length`(数组/字符串/对象)走原路 → 普通函数值 codegen 逐字节不变。解析器 `_resolveFnNode`/`_fnNameLength`:标识符→函数/类声明(`ctx.getFunction`)或 const 绑定的箭头·函数表达式·类表达式(`ctx.varInitExprs`);`obj.method`→对象字面量里该属性的函数值。`.name`=命名函数表达式自身名优先、否则绑定名/方法名、类用 id;`.length`=首个默认(AssignmentPattern)/剩余(SpreadElement)形参前的形参数(node 语义)。**全形态 g1 与 node 逐字节相等**:函数声明、箭头(名取绑定)、匿名+命名函数表达式、对象简写方法与箭头值属性、类(名+构造器 arity)、默认/剩余 `.length`。无误报(数组/字符串/数据属性/对象内数组走原路)。gen1==gen2==gen3(13615120→13631504,3 试稳定)、4 目标全交叉编译、fixtures 261/2、matrix 19/19。fixture `es/function-name-length`。**未反射**:运行时传递的函数值(参数/更深成员链,非静态可知)仍 undefined/0——需闭包属性容器结构改造(与 `fn.x=1`、custom-tag `.raw` 同一 blocker,值得一次性做侧表解锁三者)。

## 4. 数值与打印偏差(专项)

- **NaN 与 NaN-boxing 冲突**(#25/#26 修复中):NaN 位型与 boxed-int0 语义分裂。`0/0` 打印 `NaN`、typeof 正确(#25 已修 gen1 挂死),但 **`NaN !== NaN` 判定失效**(返回 false),`Number.isNaN(NaN)` → false,全局 `isNaN` 错值。
- **浮点打印精度**:15 位有效数字舍入+去尾零(2026-07-10 落地);**第 16 位边角与 node 不一致**(`0.1+0.2` → `0.3` 而非 `0.30000000000000004`;`1/3` 少一位)。shortest round-trip 未实现(ROADMAP E3)。
- **超大数字面量/打印**:`1e21` 打印为 `9223372036854775807`(int64 饱和),无科学计数法输出路径。
- **数组 ToPrimitive 缺失**:`[] + 1.5` → `1.`、`[] + "x"` → 空串(node: `1.5` / `x`)。装箱边角属 ROADMAP E2。

## 7. 一句话结论

JSBin 的 ES 支持呈「工作集深、全集浅」形态:自举链路覆盖的核心子集(闭包/class/ESM/容器/字符串/async 基础/JSON)达到生产级验证强度,但其外围存在大量运行时崩溃与比崩溃更危险的静默错值;距完整 ES,最大的结构性缺口是原型链元操作体系、RegExp 高级特性与模块动态语义,收敛路径以 ROADMAP E1–E4 为准。
