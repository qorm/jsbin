## v0.2.2 (**test262 S1 first batch (20.55% → 22.25%) + self-compile −19% + a serial bootstrap gate — gen1==gen2==gen3 byte-identical, fixtures 362/362, test262 22.25%**. **test262 S1 batch 1** — five parser/compiler fixes that flip COMPILE_FAIL **601 → 104** and add **+110 PASS (1328 → 1438 / 6462, 20.55% → 22.25%)** of the stride-5 subset: (1) **lexer Unicode escapes in identifiers** — `\u{...}` / `\uNNNN` now decode inside identifiers via a fast path (no-escape → raw slice, zero self-compile overhead) and a slow path; an escape that spells a reserved word stays a keyword and is rejected in binding position, matching Node; (2) **non-declarative for-of/in left-hand side** — `for (a of x)` / `for ([a,b] of x)` / `for (a in x)` now parse (a top-level binary `in` immediately followed by `)` is decomposed into the for-in LHS/RHS, so no core Pratt-loop change is needed; array/object expression LHSs are reinterpreted as patterns by the compiler); (3) **array rest target as a binding pattern** — `[...[x]]` / `[...{length}]` / `function f([...[y]])` (object rest deliberately stays identifier-only, per the ES `ObjectRestProperty = ...BindingIdentifier` grammar); (4) **pure/trailing elision in array patterns** — `[,]` / `[a,,]` / `[[,x]]` (the elision branch no longer double-consumes the closing `]`); (5) **object destructuring triggers getters** — `var {a}=o` / `function f({x})` / `{[k]:v}` now call `_maybe_getter` after each `_object_get`, so accessors run instead of binding the raw getter-marker garbage (direct member access was already correct via the IC's fused `_maybe_getter`). **Data-driven finding recorded**: parser-gap fixes convert COMPILE_FAIL→runnable cleanly, but single value-bug fixes convert poorly (the getter fix is correct yet flips 0 tests — test262 cases are typically multi-blocked); the remaining CRASH bucket is dominated by **non-generic array methods** (`Array.prototype.indexOf.call(new String(...))` / `.call({length,...})` SIGSEGV — array methods assume a real 0x7FFE receiver), mapped as the next architectural lever. **Performance** (self-compile **25.5 → 20.7 s user, −19%**): **prototype-key IC (stage A3.5)** — read IC sites grow to 24 B `{obj_shape, holder, index}`; a shape hit on a direct prototype key proves the slot after a holder-pointer + key self-verify (deep chains fall back uncached), with class method/field/constructor keys unified to the 0x7FFC-interned key world (a probe found bare-pointer class keys were defeating all key-identity compares, ~12% → ~6% slow-path cluster); plus **`addString`/`registerRuntimeString` interned lookup** (has+get → single get, −4%), **`resolveModulePath` compile-time cache** (resolveImports cluster 6.7% → 3.5%), and **`_strict_eq` string fused compare** (Map-key probe drops 2×strlen+memcmp for heap-string O(1) length-equality / data-segment NUL-walk early-exit, cluster 6.5% → 1.9%). **Infra**: `scripts/bootstrap-gate.sh` — a serial (mkdir-based flock) gate running the full `node→gen1→gen2→gen3` chain with `rm -f gen1 gen2 gen3` first, byte-comparing `gen1==gen2==gen3`, and asserting fixtures ≥362 with FAIL==0. **Docs**: README (EN/zh) self-hosting claims corrected to the current ARM64-only status (x64 three-target bootstrap regressed at the v1.1.0 snapshot; devirt disabled). Verified: gen1==gen2==gen3 byte-identical, fixtures 362/362, test262 22.25%.)

## v0.2.1 (**Shape (hidden-class) IC infrastructure + the self-host "layout cliff" root-cause fix — gen1==gen2==gen3 byte-identical, fixtures 362/362, test262 20.55%**. **Object shapes, stages A1–A3.1** (`docs/SHAPE_IC_DESIGN.md`): object headers grow 48→56 bytes with a new `shape_ptr@48` slot (0 = shapeless). Object literals with fully-static data keys (no spread / accessors / computed / duplicate / `__proto__` keys) and class instances (field order flattened over the inheritance chain, shadowing deduplicated, enforced by a post-constructor `count==key_count` runtime check) receive static data-segment shape descriptors (`{key_count}` — the IC only ever compares descriptor *identity*). The property read/write inline caches (`_object_get_ic` / `_object_set_ic`) move from an 8-byte index slot to a 16-byte `{cached_shape, cached_index}` dual-mode site: a shape-pointer hit proves the entire key sequence, so the count/props-ptr guards drop out (the key self-verify cmp is retained as the safety net — a probe showed removing it buys ~0 anyway, and it also keeps the G-M-P 16-byte torn-pair race benign); shapeless objects keep the byte-identical legacy path. Key-set mutations (append / delete / integer-key reorder / `setPrototypeOf`) zero the shape. **Performance is intentionally unchanged at this stage** (bench/prop ~8.7× vs Node; self-compile ~28 s native / 2.6 s node-driven — the payoff is the shape-keyed method IC this infrastructure enables next). **"Layout cliff" root cause found and fixed**: the intermittent `gen1≠gen2` fixed-point failures with page-aligned trailing zero bytes were *not* a heap-layout miscompile — macOS `writeFileSync` opened with Linux O_* constants, so `O_TRUNC` never took effect and a previous, larger output file's tail survived verbatim. The O_* constants are now per-platform (macOS `O_APPEND=0x8` works as a bonus); fixed-point discipline gains `rm -f gen1 gen2 gen3` before chains; the v1.x-era "~240 KB TDZ-string divergence" is suspected to be the same artifact. Also fixed: an 8-byte heap overrun in the hand-written classinfo header (48-byte allocation with a `shape_ptr` store at +48, A1 follow-up). A native profiling pipeline landed (macOS `sample(1)` + `LABEL_MAP` symbolization) — hotspot map recorded in plan.md (compiler driver ~15%, property slow-path cluster ~12%, module-function cluster ~17%, string internals ~8%). fixtures 362.)

## v0.2 (**TypedArray constructor globals + dynamic method dispatch — test262 TypedArray area rescued from 0%; CLI/CI repairs; .h declaration-layer decision**. The big one: **TypedArray/ArrayBuffer constructors are now real global values** — previously `new Int8Array(...)` was compile-time special-cased and the constructors never existed as values, so `ArrayBuffer.prototype.resize` in `harness/testTypedArray.js:106` read a property of undefined and threw at include load, zeroing the whole 288-test TypedArray area of test262. Constructors now materialize as 24-byte closures (`{magic, fnptr=_ta_ctor_tramp, type@16}`) with per-constructor `.prototype` singletons (`ArrayBuffer.prototype` is an empty object, so the include's resizable/immutable feature probes safely return undefined); `new TA(...)` via variables routes through a dedicated `_ta_construct` (no instance semantics), covering length/array/array-like/typed-source/ArrayBuffer-view forms. **Dynamic TypedArray method dispatch**: `new TA()` on an unbound callee now infers UNKNOWN (was OBJECT, which dropped TA instances into the generic object path), and `join`/`arrayOnly`/`ambiguous (slice/at/indexOf/includes)`/`set`/`subarray` member calls on unknown receivers tag-dispatch by header byte to the `_ta_*` implementations — with two correctness fixes found by the gates: the header-byte check must only fire on raw pointers (a 0x7FFC string's first content byte, e.g. `'A'=0x41`, masquerades as a TA type byte — broke base64/URLSearchParams/util fixtures and the self-compile), and receivers are evaluated exactly once (`_evalOnceToIdent` — mutating methods like `copyWithin`/`reverse` previously ran twice in chained calls such as `ta.copyWithin(0,1).join()`). Also: `compileArrayMap` no longer boxes TypedArray results as 0x7FFE plain arrays (the plain-array printer read `buffer@24` as `data_ptr` → SIGSEGV). test262 TypedArray area: **0/288 → 10/288 PASS** (overall 20.40% → see report; remaining area gaps: %TypedArray% proto chain, method-value reads, descriptor reflection). **CLI/CI**: `--version`/`-v` now prints `asm.js 0.1`; `resolveModulePath` no longer mistakes a `.js`-suffixed **directory** (like the renamed repo checkout `asm.js`) for a file (broke GitHub Actions builds); the Release workflow builds `asm.js-<target>` artifacts; full jsbin→asm.js/asmjs identifier sweep (417 sites, incl. `__jsbin_err`→`__asmjs_err` with all string-name readers). **Governance note**: the known bootstrap "layout cliff" (plan.md risk #1) was observed firing nondeterministically this cycle — `gen1==gen2==gen3` flips pass/fail across runs on identical sources at baseline commits as well (heap-layout-luck in the compiled compiler, pre-existing, root cause still open). This entry's verification: fixtures 362/362, fixpoint chain green in repeated runs (3/3 in the good window), TA repros byte-identical vs Node.)
